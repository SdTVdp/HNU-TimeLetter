# 数据模型 (Data Model)

> **Single Source of Truth**: 所有已落地的领域类型以 [`src/lib/types.ts`](../../src/lib/types.ts) 为准，本文档负责主体字段、来源映射与聶合规则的文字解释。页面渲染层补充模型（如 `CreationEntry` / `CreationCard`）详见 [创作公示板](../design/创作公示板.md)。

## 核心实体

### 1. Story (故事)
对应飞书多维表格中的一行记录。

| 字段名 | 类型 | 说明 | 对应飞书字段 |
| :--- | :--- | :--- | :--- |
| `id` | string | 唯一标识符 | 自动生成/记录ID |
| `characterId` | string | 角色 ID | `角色ID` |
| `characterName` | string | 角色名称 | `角色名` |
| `avatarUrl` | string | Q版头像 URL | `头像OSS_URL` (优先) 或 `头像` |
| `mainImageUrl` | string | 高清大图 URL | `大图OSS_URL` (优先) 或 `大图` |
| `content` | string | 故事文本 | `故事内容` |
| `author` | string | 投稿人/作者 | `投稿人` |
| `date` | string | 日期 (YYYY.MM.DD) | `日期` |
| `locationId` | string | 关联地点 ID | `地点ID` |

### 2. LocationPoint (地点)
前端聚合使用的地点数据结构。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | string | 地点唯一标识 (如 `lib-001`) |
| `name` | string | 地点名称 (如 `图书馆`) |
| `x` | number | SVG 横坐标百分比 (0-100) |
| `y` | number | SVG 纵坐标百分比 (0-100) |
| `stories` | Story[] | 该地点下的所有故事集合 |

### 3. CreationIdea (创作灵感记录)
对应飞书“揭示板”中的一条原始记录。

> 说明：2026-04-17 已读取飞书表结构。当前远程表名为 `揭示板`，目标视图为 `收集结果`。该记录模型尽量保持原始字段语义，不额外引入无关衍生字段。

| 字段名 | 类型 | 说明 | 对应飞书字段 |
| :--- | :--- | :--- | :--- |
| `id` | string | 灵感唯一标识符 | `record_id` |
| `cardId` | string | 业务编号 / 展示编号 | `CardID`，为空时回退 `自动编号`，仍为空时回退 `record_id` |
| `author` | string | 展示署名 | `你的昵称`，为空时回退 `提交人` |
| `submitter` | string | 飞书提交人 | `提交人` |
| `tags` | string | 单个胶囊标签值 | `请选择你要添加的内容（该表可重复提交，如需填写多项，请再次提交）` |
| `content` | string | 灵感正文 | `文本`（参考图场景下也用于承载回填的 OSS URL） |
| `images` | string[] | 附件图片 URL 列表 | `请上传你的图片`（下载后上传 OSS） |
| `createdAt` | string | 提交时间 | `提交时间` |

### 4. CreationEntry (创作卡片堆叠项)
页面渲染层使用的“卡片内堆叠条目”，是由 `CreationIdea` 投影得到的展示子集。

> 说明：`CreationEntry` 只用于 `/creation` 页面渲染，不参与同步层落盘；相较 `CreationIdea`，省去了 `submitter` 字段，其余字段与语义保持一致。详见 [创作公示板 §8.2](../design/创作公示板.md#82-entry-级模型)。

| 字段名 | 类型 | 说明 | 当前来源 |
| :--- | :--- | :--- | :--- |
| `id` | string | 记录唯一标识 | `CreationIdea.id` |
| `cardId` | string | 所属卡片编号 | `CreationIdea.cardId` |
| `author` | string | 展示署名 | `CreationIdea.author` |
| `tags` | string | 单个胶囊标签值 | `CreationIdea.tags` |
| `content` | string | 正文内容 | `CreationIdea.content` |
| `images` | string[] | 图片内容 | `CreationIdea.images` |
| `createdAt` | string | 提交时间 | `CreationIdea.createdAt` |

### 5. CreationCard (创作卡片)
对应 `/creation` 页面瀑布流中的一张卡片。

> 说明：`CreationCard` 不是飞书中的单条记录，而是将所有 `CardID` 相同的 `CreationIdea` 聚合、投影为 `CreationEntry` 后得到的页面实体。`CardID` 的重复是设计要求，不应在同步阶段被去重丢弃。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| `id` | string | 卡片唯一标识，使用 `CardID` |
| `cardId` | string | 当前卡片对应的 `CardID` |
| `addIdeaUrl` | string | 卡片右上角 `新增创意` 按钮链接 |
| `entries` | `CreationEntry[]` | 卡片内部堆叠的所有记录项 |

## 数据流向

1. **飞书故事表**：运营人员录入首页展陈故事数据。
2. **飞书创作公示板表**：运营人员录入共创灵感数据。
3. **同步服务**（[`src/lib/sync-service.ts`](../../src/lib/sync-service.ts)）：
   - 故事同步链路生成 [`src/data/content.json`](../../src/data/content.json)。
   - 创作公示板同步链路生成 [`src/data/creation-board.json`](../../src/data/creation-board.json)，并在参考图场景下将 OSS URL 回填到飞书 `文本` 字段。
   - CLI 包装 [`src/scripts/sync-feishu.ts`](../../src/scripts/sync-feishu.ts) 和 `/admin` 手动同步 API 同享同一入口。
4. **Frontend**：
   - 首页 `/` 读取 `content.json`。
   - 创作公示板 `/creation` 读取 `creation-board.json` 中的记录级数据，由 [`src/components/creation/utils.ts`](../../src/components/creation/utils.ts) 在页面层按 `CardID` 聚合为 `CreationCard`。

## 状态管理
全局状态由 Zustand（[`src/lib/store.ts`](../../src/lib/store.ts)）管理：
- `isEnvelopeOpened`：信封是否已拆开。
- `isTransitioning`：开信 → 主体验的全屏过渡遮罩状态。
- `selectedLocationId`：当前选中的地点。
- `currentStoryIndex`：当前阅读的故事索引。
