# 技术架构与技术栈

## 核心技术栈
- **Framework**: Next.js 16.1.6 (App Router)
- **Language**: TypeScript 5+
- **Styling**: Tailwind CSS v4
- **State Management**: Zustand v5
- **Animation**: Framer Motion v12（开屏、转场、卡片与滚动区块动效）
- **Smooth Scroll**: Lenis（桌面端启封态下滚页面群）
- **UI Components**: Shadcn/UI (Radix UI) + Lucide React
- **Build Tool**: Turbopack (Next.js default)

## 外部服务
- **CMS**: 飞书多维表格 (Feishu Bitable) - 作为唯一数据源。
- **Storage**: 阿里云 OSS (Aliyun Object Storage Service) - 存储图片资源。
- **Deployment**: Vercel (推荐) / Static Export。

## 工程约束
- **TypeScript Compile Scope**: `tsconfig.json` 已显式排除 `agents/`，避免协作代理资产进入应用编译范围并影响 `npm run build`。
- **Collaboration Assets**: 根目录 `agents/` 与 `.opencode/agents/` 用于代理、技能与知识库协作，不属于前端运行时代码。
- **Local Tooling**: 根目录 `opencode.json` 用于注册项目级 Playwright MCP server，属于本地开发辅助配置，不参与线上运行。

## 目录结构规范
```text
src/
├── app/               # Next.js 路由页面
├── components/
│   ├── ui/            # Shadcn 基础组件 (禁止直接修改逻辑)
│   ├── shared/        # 通用业务组件 (Envelope, Loading)
│   ├── desktop/       # [Dev B] PC 端专用组件
│   └── mobile/        # [Dev C] 移动端专用组件
├── lib/               # 工具函数、类型定义 (types.ts)、Store
├── data/              # 生成的数据文件 (content.json)
└── scripts/           # 数据同步脚本 (sync-feishu.ts)
```

仓库根目录另含 `agents/`、`.opencode/agents/`、`.trae/skills/` 等协作资产目录；这些目录服务于开发流程，不纳入 `src/` 运行时代码边界。
