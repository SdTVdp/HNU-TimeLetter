## P0（需立即修复）

### 1) `Story` 类型重复定义且不一致
- 问题：`lib/types.ts` 已有 `Story`，但 `MobileExperience.tsx`、`MobileDetailModal.tsx`、`StoryFeed.tsx` 各自重复声明，且字段不统一（移动端多 `locationName?: string`）。
- 风险：类型漂移、维护成本上升、潜在运行时数据不一致。
- 建议：
  1. 在 `lib/types.ts` 补充 `locationName?: string`。
  2. 三个组件统一 `import type { Story } from '@/lib/types'`。

### 2) 图片组件使用废弃 API
- 问题：`InteractiveMap.tsx`、`StaticMapModal.tsx` 使用 `onLoadingComplete`（Next.js 新版本已废弃）。
- 风险：升级兼容性差，未来版本可能失效。
- 建议：改为 `onLoad={(e) => { const img = e.currentTarget; ... }}`。

## P1（重要优化）

### 4) Zustand Store 存在死代码
- 问题：`selectedLocationId`、`currentStoryIndex` 未被组件消费。
- 风险：状态来源分散，误导后续维护。
- 建议：删除未使用状态，或统一将相关状态迁移进 Store。
- 状态：✅ 已修复（2026-03-05，`src/lib/store.ts` 已删除未使用状态字段，仅保留 `isEnvelopeOpened`）。

### 5) 地图尺寸计算逻辑重复
- 问题：`InteractiveMap.tsx` 与 `StaticMapModal.tsx` 各维护一套近似 `ResizeObserver + mapAspect` 逻辑。
- 风险：修复和迭代需要双改，容易不一致。
- 建议：抽取 `useMapSize(mapAspect)` 复用。
- 状态：✅ 已部分修复（2026-03-05，`InteractiveMap.tsx` 已合并重复尺寸监听，将 `rollOffset` 与 `mapSize` 计算统一到同一 `ResizeObserver` 回调；`StaticMapModal.tsx` 暂未抽离到共享 Hook）。

### 6) `MobileExperience.tsx` 中 `findIndex` 重复执行
- 问题：`locationStories.findIndex(...)` 在渲染中多次调用。
- 风险：不必要的重复计算。
- 建议：用 `useMemo` 缓存索引结果。
- 状态：✅ 已修复（2026-03-05，新增 `currentLocalIndex`、`currentLocationIndex` 两个 `useMemo` 缓存并复用到导航与渲染）。

### 7) `scripts/sync-feishu.ts` 体积过大（553 行）
- 问题：单文件承担过多职责。
- 风险：可读性与测试性下降。
- 建议：按职责拆分到 `scripts/services/*`（解析、上传、映射、写回等）。

## P2（常规改进）

### 8) 噪点纹理依赖外部 URL 且多处重复
- 问题：`https://grainy-gradients.vercel.app/noise.svg` 在代码中重复引用。
- 风险：第三方不可用时样式降级。
- 建议：下载到 `public/images/noise.svg`，通过全局样式统一引用。

### 9) `EnvelopeIntro.tsx` 动画时序由 `setTimeout` 硬编码
- 风险：与动画参数耦合，后续改动易错。
- 建议：使用 Framer Motion 控制器 Promise（`controls.start()`）串联动画。

### 10) `page.tsx` 首帧空白门控
- 问题：`mounted === false` 渲染空容器。
- 风险：白屏与 CLS 体验问题。
- 建议：优先 CSS 媒体查询分流，减少 JS 门控。

### 11) `MobileDetailModal` 拖拽回弹体验生硬
- 问题：`dragY.set(0)` 硬跳。
- 建议：改为 `animate(dragY, 0, { type: 'spring', stiffness: 300, damping: 30 })`。

### 12) `StoryFeed.tsx` 的 `allStories` 未缓存
- 问题：每次渲染都 `flatMap`。
- 建议：`useMemo` 缓存派生数据。

### 13) `not-found.tsx` 使用 shadcn token 但全局变量可能未定义
- 风险：样式在部分环境失效。
- 建议：统一 token 体系，或改为项目已定义样式类。

### 14) 缺少 CI 门禁
- 问题：未见自动化 `lint` / `tsc --noEmit`。
- 建议：增加 CI 工作流，在 PR 阶段阻断明显质量问题。
