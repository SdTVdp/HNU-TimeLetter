# 视觉设计规范 (Design System)

> **版本号**: v2.0.0
> **状态**: 2026-04-15 共创中 / 作为前端设计基线
> **本基线锁定**: 背景色 `#ece9e4`、主色 `#c23643`、标题字体 `public/ChillDINGothic_SemiBold.otf`、正文字体 `public/ZouLDFXKAJ.ttf`
> **排除方案**: 「海大蓝 + Noto 字体」不属于本规范，任何界面均不得以其作为实现依据。

## 1. 视觉方向

- **核心气质**: 纸本情书、火漆印章、海风旧影、编辑感留白。
- **视觉目标**: 页面以「被翻开的纪念册」与「收到的私人来信」为意象，避免「校园信息站」风格。
- **材质语言**: 以暖灰纸面作为主舞台，暗红色负责情绪锚点，铜金色仅用于细节提气，不抢主色。
- **构图原则**: 大留白、强标题、少而准的强调色、局部纹理，不做大面积纯科技蓝/纯白面板。

## 2. 全局设计变量 (Design Tokens)

### 2.1 色彩体系 (Color Palette)

- **Background / 页面主背景**: `#ece9e4`
  - 用于页面底色、全局基底、超大面积留白区域。
- **Surface / 纸面卡片**: `#f6f1eb`
  - 用于卡片、模态、抽屉、内层信纸。
- **Surface Strong / 分区底色**: `#e2d7cf`
  - 用于次级区块、卷轴、时间轴、悬浮条。
- **Primary / 主强调色**: `#c23643`
  - 用于主按钮、选中态、当前故事、关键 CTA、链接高亮。
- **Primary Deep / 主色悬浮态**: `#a92d38`
  - 用于主按钮 hover、按下态、深色火漆印边缘。
- **Primary Soft / 主色浅底**: `#ecd0d4`
  - 用于标签背景、轻提示、选中态底色。
- **Ink Strong / 标题深墨色**: `#342628`
  - 用于大标题、段首强调、重要说明。
- **Ink / 正文默认色**: `#5a4748`
  - 用于正文、说明文、组件内容文本。
- **Ink Muted / 次级文本色**: `#7a6664`
  - 用于日期、地点、元信息、说明性标签。
- **Border / 边框色**: `#cbbdb5`
  - 用于卡片描边、输入框边界、分隔线。
- **Accent Metal / 点缀金属色**: `#a9825e`
  - 用于邮票边缘、火漆印金属反光、装饰线、徽章描边。
- **Focus Ring / 焦点态**: `rgba(194, 54, 67, 0.28)`
  - 用于键盘焦点、可访问性交互轮廓。

### 2.2 字体排印 (Typography)

- **Headings / 标题字体**: `public/ChillDINGothic_SemiBold.otf`
  - 用于站点标题、章节标题、按钮主文案、英文标签。
- **Body / 正文字体**: `public/ZouLDFXKAJ.ttf`
  - 用于正文段落、故事文案、说明区域。
- **Fallback 策略**:
  - 中文标题 fallback 使用 `PingFang SC`, `Microsoft YaHei`, `sans-serif`。
  - 正文字体 fallback 优先回退到标题字体，再回退系统中文字体。

#### 2.2.1 响应式字号变量

- **`--text-h1`**: `clamp(2.5rem, 5vw + 1rem, 5rem)`
  - 用于页面主标题与核心视觉标题。
- **`--text-h2`**: `clamp(2rem, 3.5vw + 0.5rem, 3.5rem)`
  - 用于重要章节标题与大区块分层。
- **`--text-h3`**: `clamp(1.5rem, 2.5vw, 2.5rem)`
  - 用于常规模块标题与次级标题。
- **`--text-intro`**: `clamp(1.125rem, 1.8vw, 1.75rem)`
  - 用于导语、摘要与大正文内容。
- **`--text-base`**: `clamp(1rem, 0.8vw + 0.5rem, 1.125rem)`
  - 用于标准正文、长篇阅读与页面默认文字基线。

#### 2.2.2 全局排版基线

- **HTML 渲染优化**:
  - 开启 `-webkit-font-smoothing: antialiased` 与 `-moz-osx-font-smoothing: grayscale`，提升高分屏文字平滑度。
  - 统一 `text-size-adjust: 100%`，避免移动端横竖屏切换时出现系统自动放大字体。
- **Body 基线**:
  - 默认字号使用 `var(--text-base)`。
  - 默认行高使用 `1.8`。
  - 默认外边距为 `0`。

#### 2.2.3 标题排版规则

- **`h1 / h2 / h3` 通用规则**:
  - 行高统一为 `1.4`。
  - 字重统一为 `400`。
  - `margin-top` 统一为 `0`。
  - `margin-bottom` 统一为 `0.8em`。
  - 字距统一为 `0.08em`。
- **`h1`**:
  - 字号使用 `var(--text-h1)`。
- **`h2`**:
  - 字号使用 `var(--text-h2)`。
- **`h3`**:
  - 字号使用 `var(--text-h3)`。

#### 2.2.4 正文排版规则

- **介绍正文 / `.text-intro`**:
  - 字号使用 `var(--text-intro)`。
  - 行高使用 `1.7`。
  - 字重使用 `400`。
  - 字距使用 `0.05em`。
  - 最大宽度限制为 `800px`，避免桌面端单行过长。
  - 段后间距使用 `3rem`。
  - 适用于导语、章节引入与需要更强留白感的大正文内容。
- **阅读正文 / `.text-reading`**:
  - 字号使用 `var(--text-base)`。
  - 行高使用 `2`，优先保障长文阅读呼吸感。
  - 字重使用 `300`。
  - 字距使用 `0.03em`。
  - 最大宽度限制为 `65ch`，控制单行字符数。
  - 段后间距使用 `2rem`。
  - 适用于详情说明、长篇叙述与连续阅读内容。

#### 2.2.5 使用约束

- 排除排版组合：更激进的大标题尺度、标题 `800` 字重、负字距、标题 `1.1` 行高、导语 `1.5` 行高、阅读正文 `1.8` 行高及同类段后间距组合均不得作为排版依据。
- 标题、导语、正文与说明样式统一从 `--text-h1`、`--text-h2`、`--text-h3`、`--text-intro`、`--text-base` 中选取，不引入额外字号变量。
- 标题层级统一遵循轻字重、宽字距与更大留白，单个标题层级不附加额外的字距规则。
- 正文样式统一以 `.text-intro` 与 `.text-reading` 为基线，导语采用 `400` 字重、阅读正文采用 `300` 字重。

### 2.3 圆角、阴影与层次

- **Base Radius / 基础圆角**: `20px`
- **Panel Radius / 卡片圆角**: `28px`
- **Modal Radius / 模态圆角**: `32px`
- **Pill Radius / 标签圆角**: `999px`
- **Card Shadow / 常规卡片阴影**: `0 24px 60px rgba(69, 39, 40, 0.12)`
- **Card Hover Shadow / 悬浮卡片阴影**: `0 32px 80px rgba(69, 39, 40, 0.16)`
- **Inset Highlight / 纸面内高光**: `inset 0 1px 0 rgba(255,255,255,0.56)`
- **Border Opacity**: 常规描边不低于 `14%`，避免在纸色背景上完全消失。

### 2.4 动效与纹理

- **页面入场**: 使用 `600ms-900ms` 的淡入、轻微上移、模糊消散组合。
- **悬浮反馈**: `140ms-180ms`，以位移、阴影和亮度变化为主，不使用强弹跳。
- **玻璃/模糊**: 模态与悬浮层允许 `12px-18px` 背景模糊，但透明度不高于 `0.82`。
- **纸张纹理**: 推荐 `6%-10%` 可见度的纸纹或噪点，不盖住正文。
- **高光控制**: 页面级别可有柔和径向光，但不要用强烈彩虹渐变或科技感霓虹。

## 3. 组件级设计建议

### 3.1 页面主底 (Page Canvas)

- 背景不是单一纯色，应由暖灰底色 + 柔和径向高光 + 极轻纸纹三层组成。
- 页面最底层统一使用 `Background`，局部容器再叠 `Surface` 与 `Surface Strong`。

### 3.2 按钮与交互控件

- **主按钮**: 背景 `#c23643`，文字使用接近白纸色 `#fff6f4`。
- **主按钮 hover**: 切到 `#a92d38`，并抬升 `2px-4px`。
- **次按钮**: 使用 `Surface` 底色 + `Border` 描边，hover 时背景向 `Primary Soft` 轻微偏移。
- **链接**: 默认与主色一致，hover 不下划整段，而是用底部细线或颜色加深。

### 3.3 卡片、卷轴与纸面容器

- 大卡片统一采用“纸面层”表达，不直接用纯白。
- 模态和故事卡文本区需保留纸感内阴影与轻边框，避免漂浮得像默认组件库。
- 卷轴、时间条、地图侧边带建议使用 `Surface Strong`，并辅以铜金细纹。

### 3.4 标签、徽章与元信息

- 标签背景优先使用 `Primary Soft` 或 `Surface Strong`。
- 标签文字优先使用 `Ink Strong` 或 `Primary Deep`。
- 日期、地点、栏目英文副标题建议使用标题字体，不与正文混排成一片。

### 3.5 邮票、Pin 点与火漆细节

- 邮票边框和地图 Pin 可引入 `Accent Metal` 作为局部描边，增强「收藏品」感。
- 火漆印、当前地点高亮、关键操作的激活态统一使用主色；全站仅留用这一项抢眼强调色。

## 4. 实现映射约束 (Implementation SSOT)

- `src/app/layout.tsx`
  - 负责加载本地字体，并将 `font-serif` 映射为标题字体、`font-sans` 映射为正文字体。
- `src/app/globals.css`
  - 作为设计 token 单一事实来源 (SSOT)，颜色、圆角、阴影、基础纹理均在此维护。
- 写入新样式时优先使用语义 token：
  - `bg-background`
  - `bg-card`
  - `text-foreground`
  - `text-muted-foreground`
  - `bg-primary`
  - `border-border`
- 代码库中禁止使用 `text-stone-*`、`bg-[#xxxxxx]` 这类脱离设计 token 体系的写法。

## 5. 关键视觉资产清单 (Assets Checklist)

### 5.1 开场信封 (Envelope Intro)

为了实现“开信”动画与关于页过渡，需要：

- [ ] **信封组件**: 背面、盖子、信纸、火漆印分层资源。
- [ ] **关于页面**:
  - 页面背景纹理（纸张/水彩/旧照片边缘）。
  - 大标题设计：“与她的海大时光笺”（SVG/高清 PNG）。
  - 团队成员头像/展示图。

### 5.2 海大地图 (The Map)

- **PC 端**:
  - 格式: SVG（最佳）或高清 PNG（宽 >= 2560px）。
  - 风格: 偏手绘、编辑插画感或低饱和扁平风，不建议科技蓝地图。
  - 注意: 四周预留背景色，防止裁切。
- **移动端**:
  - 格式: 竖版/方形静态图（仅供查看）。

### 5.3 邮票与标记 (Stamps & Pins)

- **地图坐标点 (Pin)**:
  - Q 版头像边框容器（`48x48` 或 `64x64`）。
  - 激活态允许叠加主色描边与轻外发光。
- **邮票样式 (Stamp Frame)**:
  - 齿孔边框素材（中间镂空）。
  - 宽高比固定（`3:4` 或 `1:1`）。

### 5.4 PC 端时光卷轴 (Scroll Strip)

- **卷轴主体**:
  - 背景: `linear-gradient(to right, #ddd1c6 0%, #ede3da 58%, rgba(237,227,218,0.78) 100%)`
  - 宽度: 固定 `56px`
- **阴影与厚度**:
  - 外部投影: `6px 0 28px rgba(79,53,54,0.16)`
  - 内部阴影: `inset -2px 0 0 rgba(138,113,92,0.18)`
- **装饰元素**:
  - 卷纸横纹: `rgba(154,122,97,0.16)`
  - 交互指引: “展开地图”竖排文字默认使用 `Ink Muted`，激活箭头可使用 `Primary`

### 5.5 PC 端故事卡片堆 (Story Card Stack)

- **层级**: `z-index: 30`，高于卷轴层，支持卡片覆盖卷轴。
- **卡片**: 保持图片主导，但边缘和说明层必须与系统纸面风格统一。
- **文本区**:
  - 居中布局，默认隐藏。
  - 包含静态邮票装饰。
  - 背景使用 `Surface`，允许轻纸张纹理与内高光。

### 5.6 移动端卡片流

- **封面图**: 统一比例（`4:3` 或 `16:9`）。
- **详情页**: 上图下文，标题与正文保持清晰的字体对比。
- **FAB**: 右下角悬浮地图按钮，使用主色，不引入蓝紫渐变。

## 6. 交付格式要求

1. **设计稿**: Figma（Dev Mode）或蓝湖。
2. **切图**:
   - 图标/Logo/线条 -> **SVG**
   - 照片/插画 -> **WebP** / **PNG**（@2x/@3x）
3. **命名**: `component_state.format`（例如 `stamp_frame_active.png`）
