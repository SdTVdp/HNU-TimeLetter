# 后台管理系统设计文档 (Admin System Design)

> **版本**: v1.1.0
> **状态**: 已定稿
> **作者**: Dev Team
> **受众**: 内部开发人员

## 1. 项目概述 (Overview)

为了简化《与她的海大时光笺》项目的运维流程，我们需要构建一个轻量级的后台管理系统。该系统集成在现有 Next.js 项目中，主要服务于开发和运维人员，提供数据同步控制、定时任务管理、系统状态监控以及内容数据查看功能。

**核心需求**:
1.  **Web 界面操作**: 在浏览器中直接管理定时任务（开关、Cron 表达式）。
2.  **简单鉴权**: 采用单一口令验证机制。
3.  **环境适配**: 适配服务器部署环境（Node.js 进程内调度）。
4.  **数据可视化**: 查看 `content.json` 中的地点和故事数据。

## 2. 系统架构 (System Architecture)

### 2.1 技术选型
- **前端框架**: Next.js App Router (`/admin` 路由)
- **UI 组件**: Shadcn UI (复用现有组件库)
- **后端服务**: Next.js API Routes (`/api/admin/*`)
- **定时任务**: `node-schedule` (支持类 Cron 表达式，随 Node.js 服务启动)
- **数据存储**: 本地 JSON 文件 (`config/admin.json`) 用于存储配置和简单的日志，无需额外数据库。

### 2.2 目录结构规划
```text
src/
├── app/admin/              # 后台页面
│   ├── login/              # 登录页
│   ├── dashboard/          # 控制台 (包含监控与操作)
│   ├── content/            # 内容查看页
│   └── layout.tsx          # 后台布局 (Sidebar, AuthCheck)
├── app/api/admin/          # 后台 API Route Handlers
│   ├── login/              # 登录 API
│   ├── sync/               # 同步 API
│   └── status/             # 状态 API
├── lib/admin/              # 后台核心逻辑
│   ├── auth.ts             # 鉴权逻辑
│   ├── scheduler.ts        # 定时任务管理器
│   ├── logger.ts           # 简单文件日志工具
│   └── config.ts           # 读写 admin.json 配置
└── scripts/                # 脚本
    └── sync-feishu.ts      # 飞书同步 CLI 入口
```

## 3. 功能模块 (Modules)

### 3.1 鉴权模块 (Authentication)
- **机制**: 基于 Cookie 的简单密码验证。
- **配置**: 密码存储在环境变量 `ADMIN_PASSWORD` 中。
- **流程**:
  1. 用户访问 `/admin`，被 Middleware 拦截。
  2. 若无有效 Cookie，重定向至 `/admin/login`。
  3. 输入密码 -> API 验证 -> 设置 HttpOnly Cookie -> 跳转 `/admin/dashboard`。

### 3.2 飞书数据同步服务 (Feishu Sync Service)
- **重构**: 将现有的 `src/scripts/sync-feishu.ts` 封装为 `src/lib/sync-service.ts`，使其可被 API 和 CLI 同时调用。
- **功能**:
  - **全量同步**: 拉取飞书数据 -> 处理附件 -> 更新 `src/data/content.json`。
  - **状态反馈**: 返回同步结果（成功/失败、条数、耗时）。

### 3.3 定时任务管理 (Scheduler & Cron)
鉴于部署环境为服务器（非 Serverless），我们采用 **Node.js 进程内调度** 方案。

- **配置存储**: `src/config/admin.json`
  ```json
  {
    "sync": {
      "enabled": true,         // 开关
      "cron": "0 0 * * *",     // Cron 表达式 (如: 每天凌晨)
      "lastRun": "2023-10-27T10:00:00Z",
      "status": "idle"         // idle | running
    }
  }
  ```
- **核心逻辑**:
  - 应用启动时 (`src/instrumentation.ts` 或自定义 Server 入口) 初始化 Scheduler。
  - **Web 操作**: 后台提供界面修改 Cron 表达式和开关。
  - **热更新**: 修改配置后，API 重新加载任务，无需重启服务。

### 3.4 运维监控可视化 (Dashboard)
- **同步状态卡片**:
  - 上次同步时间
  - 同步状态 (成功/失败/进行中)
  - 下次计划执行时间
- **日志查看器**:
  - 读取 `logs/sync.log` (或内存日志) 的最后 50 行。
  - 展示同步过程中的关键信息（如：新增地点 X 个，图片上传 Y 张）。
- **手动触发**:
  - 提供 "立即同步" 按钮，点击后通过 API 触发同步任务。

### 3.5 内容数据查看 (Content Viewer)
- **功能**: 读取并可视化 `src/data/content.json` 文件内容。
- **展示形式**: 表格 (Table)。
- **层级**:
  - **地点列表**: 展示地点 ID、名称、坐标。
  - **故事详情**: 点击地点可展开查看该地点下的故事列表（角色、作者、内容预览）。
- **交互**: 支持简单的搜索和筛选。

## 4. 开发计划 (Roadmap)

1.  **Phase 1: 基础框架 & 鉴权**
    - 搭建 `/admin` 路由和登录页。
    - 实现 Middleware 鉴权。
2.  **Phase 2: 内容数据可视化** (调整顺序，优先满足查看需求)
    - 实现 `content.json` 读取 API。
    - 使用 Table 组件展示数据。
3.  **Phase 3: 同步逻辑重构**
    - 提取 `sync-feishu.ts` 为 Service。
    - 实现手动触发同步 API。
4.  **Phase 4: 定时任务与配置**
    - 引入 `node-schedule`。
    - 实现配置读写与任务热更新。
