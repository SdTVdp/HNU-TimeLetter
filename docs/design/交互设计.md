# 交互设计与页面逻辑 (Interaction Design)

本文档定义了“与她的海大时光笺”的核心交互逻辑与用户体验设计。

## 1. 全局体验 (Global Experience)

### 1.1 开场仪式 (Intro Page)
用户进入网站时的第一印象，旨在建立沉浸感并传达项目理念。当前版本保留启封主屏，并在未开信状态下于首屏下方接入“关于企划 / 关于我们 / 鸣谢 / 页脚”下滚页面群；完成开信过渡后，再切换到地图 / 集邮册主体验。

#### 启封主屏 (The Envelope)
- **视觉**: 
  - **网格**: 首屏基于五列竖向网格排版，丝带贴合第一列右边界，标题居于第二列，信封居于右侧三列的中心。
  - **标题**: 左侧采用竖排标题“与她的海大时光笺”，与整高红色丝带并置，形成首屏识别锚点。
  - **核心**: 右侧展示信封主体，火漆印使用实物质感素材，信封采用固定尺寸主视觉。
- **交互**: 
  - **点击信封**: 
    - **动画**: 火漆碎裂 -> 封盖翻开与信纸抽出 -> 信封壳体整体下落 -> 信纸移动到视口中心并放大铺满 -> 过渡遮罩淡出 -> 主界面显示。
    - **状态**: 点击后先进入 `opening` 阶段，并在信纸放大前开启 `isTransitioning` 过渡遮罩；动画完成后设 `isEnvelopeOpened = true`，再由页面层延时收起遮罩。
- **入场动效**:
  - **丝带**: 首次进入页面时基于 `clip-path` 执行自上而下的垂直裁剪显影，不改变位置与尺寸，采用双层 `requestAnimationFrame` 触发，时长 `1400ms`，缓动为 `cubic-bezier(0.16, 1, 0.3, 1)`。
  - **信封**: 丝带先显影，信封延迟约 `1500ms` 以后以 Spring 方式自上方飘落，落定后进入轻微呼吸浮动。
  - **标题**: 标题在信封落定后淡入；进入开信阶段时同步上移并轻微模糊淡出。
  - **火漆**: 火漆印在待机态保持固定 `-5deg` 倾角，不再提供 hover / tap 形变反馈。
- **滚动策略**:
  - 未开信状态下，首屏下方直接承接“关于企划 / 关于我们 / 鸣谢 / 页脚”信息页面群。
  - 桌面端在启封态下使用 Lenis 提供平滑滚动并隐藏原生滚动条；移动端保持原生纵向滚动。
  - 这些信息页面属于启封态内容层，开信完成后会整体切换到地图 / 集邮册主体验。

### 1.2 响应式分流 (Responsive Strategy)
采用 **双端差异化体验** 策略，而非简单的 CSS 响应式缩放。
- **PC 端 (>768px)**: 进入 **[时光画卷 (Desktop View)](#2-pc-端-时光画卷)**。
- **移动端 (≤768px)**: 进入 **[掌心集邮册 (Mobile View)](#3-移动端-掌心集邮册)**。
- **实现**: 在 `app/page.tsx` 中根据屏幕宽度条件渲染不同组件树。

### 1.3 独立路由扩展 (Additional Route)

- **首页 `/`**: 承载启封主屏、下滚页面群与地图 / 集邮册主体验。
- **创作公示板 `/creation`**: 承载群友共创灵感的公开浏览页，服务于“灵感沉淀与延展”，不与首页主展陈内容混用。
- **数据策略**: `/creation` 与首页一样，均基于飞书同步后的本地静态数据渲染。

## 2. PC 端: 时光画卷 (Desktop View)

> **核心隐喻**: “横向展开的时光画轴” —— 沉浸、动态、叙事。

### 2.1 交互式地图与卷轴 (Interactive Map & Scroll)
- **布局策略**: 
  - **初始状态 (Idle)**: 地图全屏显示，用户可以在地图上寻找地点。
  - **卷起动作 (Rolling)**: 点击地点 Pin 后，地图从左侧向右侧水平收纳，最终在右侧边缘形成一个宽 56px 的 **[卷轴条 (Scroll Strip)](#23-卷轴条视觉设计)**。
  - **收纳逻辑**: 地图容器使用 `overflow-hidden` 裁剪，内层地图保持 `100vw` 宽度并右锚定，确保收纳过程中 Pin 点的相对坐标依然准确。
- **层级关系 (Layering)**:
  - 为了保证交互体验，**故事面板 (z-30)** 位于 **地图/卷轴容器 (z-10)** 之上。
  - 这确保了在故事面板中拖拽卡片时，卡片可以自由覆盖在卷轴条上，不会产生视觉遮挡或被裁剪。

### 2.2 故事看板 (Story Stage)
- **出现时序**: 
  - **顺序感控制**: 故事看板不会在地图开始卷起时立即出现，而是等待地图完全收纳成卷轴（即 `rolled` 状态）后才开始进场。
  - **进场动画**: 采用从下方弹出的弹簧动画（Spring），无内部延迟，确保地图卷完后立即无缝衔接。
- **核心交互**:
  - **堆叠卡片 (Card Stack)**: 位于看板中心，支持向左/向右拖拽飞出，循环展示。
  - **底层卷轴**: 卷轴条始终位于右侧背景，作为返回地图的常驻入口。

### 2.3 卷轴条交互 (Scroll Strip Interaction)
- **还原逻辑**: 点击右侧卷轴条触发 `unrolling` 状态。
- **顺序感**: 
  1. 故事看板首先执行退场动画（向上飞出并淡出）。
  2. 看板完全退出后，地图容器开始向左展开恢复至 `100%` 宽度。
  3. 状态重置为 `idle`，激活地点清空。

## 3. 移动端: 掌心集邮册 (Mobile View)

> **核心隐喻**: “手中的集邮册” —— 高效、私密、易读。

### 3.1 故事瀑布流 (Story Feed)
- **策略**: 放弃手机端难用的地图交互，改为垂直瀑布流。
- **卡片**: 展示高清大图缩略 + 地点/角色信息。
- **FAB (悬浮按钮)**: 右下角常驻“地图”图标按钮，点击弹出一张 **纯静态海大地图** 图片，仅供查看地理位置，不可交互。

### 3.2 沉浸阅读 (Detail Modal)
- **转场**: 点击列表卡片 -> **共享元素转场 (Shared Element Transition)** -> 图片放大至全屏。
- **布局**: 上图下文。
  - **上部**: 高清大图 (占据约 50% 高度)，覆盖左右切换箭头。
  - **中部**: 角色头像与信息区。
  - **下部**: 滚动阅读故事文本。
- **手势**:
  - **点击箭头**: 切换同一地点的不同故事。
  - **左右滑动**: 切换不同地点。
  - **下拉 (Pull-to-close)**: 关闭详情页返回列表。

## 4. 下滚页面群 (Scroll-down Pages)

> **核心隐喻**：启封主屏下方向外延展的“展览走廊”——用户在未开信状态下即可继续向下浏览，完成开信后则切换到地图 / 集邮册主体验。

### 4.0 全局滚动系统

#### 平滑滚动 (Smooth Scroll)
- **桌面端策略**：启封态下由 Lenis 接管桌面端滚轮滚动，提供带惯性的平滑过渡；移动端保持原生滚动。
- **关键参数**：`duration: 1.2`、指数型 easing、`wheelMultiplier: 1`、`touchMultiplier: 2`。
- **辅助样式**：通过 `html.lenis` / `.lenis-scrolling` 等状态类隐藏原生滚动条，并处理停止态与滚动态样式。
- **实现位置**：`src/lib/useVirtualScroll.ts` 与 `src/app/globals.css`。

#### 红色引导线 (Red Guide Line)
- **视觉**：横跨三个页面的折线引导线，宽度 `100px`，颜色 `#c23643`，端点圆角（`stroke-linecap: round`, `stroke-linejoin: round`）。
- **层级**：位于背景之上、正文内容之下（`z-index: 1`）；首页 `page-paper` 会遮住起点上方延伸段。
- **路径定义**（坐标为占比，页面左上角为 `0%, 0%`）：
  - 关于企划页：P0(17.396%, -20%) → P1(17.396%, 0%) → P2(17.396%, 5%) → P3(19.15%, 7.50%) → P4(107.5%, 40.43%)
  - 鸣谢 / 页脚段：P5(-2.19%, 65.40%) → P6(32%, 150%)
- **动效逻辑**：非可逆单向线性插值（Non-reversible Linear Interpolation）
  - 使用 `Math.max(cachedMaxScroll, currentScroll)` 作为绝对进度标量
  - **向下滚动**：SVG 路径描边按比例实时正向曝光渲染
  - **向上回滚**：路径端点严格冻结在历史最远端，无反向收缩
  - **再次越过阈值**：插值计算无缝重连，继续正向曝光
- **实现**：一条连续 SVG `<path>` 依次连接 `P0 → P6`，其中 `P4 → P5` 对角跨越整个“关于我们”页面，`P6` 继续延伸进页脚红色区域。

### 4.1 关于企划 (About Project)
- **布局**：全屏（`100vh`），信纸色背景。
- **文案**：
  > 关于企划
  >
  > 这是一个聚焦于海大校园、由群友灵感驱动的 AIGC 视觉共创展。
  > 企划发起自海大 Gal同好群⌈海带姬松书院⌋。我们试图打破次元的边界，将那些原本只存在于游戏屏幕中的少女，带入触手可及的真实校园。
  > 这里的每一处选址、每一位登场人物，乃至于画面背后承载的那段微小故事，均脱胎于群友的提案与共创。由大家提供喜爱的人物与故事线索，再由制作组借由实景拍摄与 AIGC 技术将其化为现实。
  > 这不仅一次单向的画集展示，更是一场属于我们的集体记忆创作。我们以这方校园为画框，邀你一同拆开这封跨越虚实的"海大时光笺"。
- **排版**：标题使用标题字体（`font-serif`），正文使用正文字体（`font-sans`），大留白风格。

### 4.2 关于我们 (About Us)
- **布局**：全屏（`100vh`），左右分栏，并主动避让从右上到左下穿越的引导线。
- **左侧**：文案区
  > 关于我们
  >
  > 「海带视研」为本企划的策展与运营团队。主要负责收集与梳理各项提案，协调摄影及后期制作，将抽象的文字构想转化为具体的视觉展品。
- **右侧**：人物展示区
  - 约 7 个成员，每人展示：圆形头像 + 昵称 + 职责
  - 内容暂留空（占位设计）
- **排版**：左侧文案约占 `20% ~ 35%` 视宽，右侧成员网格右对齐展开，整体垂直居中对齐。

### 4.3 鸣谢 (Credits)
- **布局**：全屏，整体偏右对齐。
- **上部**：鸣谢文案区
  > 鸣谢
  >
  > 待定开发中，大概100字
- **下部**：参与贡献名单
  - 三排横向滚动昵称文字展示区域
  - 自动无限滚动（marquee 风格），各排速度/方向可有差异
- **引导线**：P5(-2.19%, 65.40%) → P6(32%, 150%) 经过此页面底部区域，并继续延伸进页脚红色区域。

### 4.4 页脚 (Footer)
- **动效**：揭露式视差动画（Reveal Parallax）
  - **触发**：页脚固定在页面根层，正文滚动到占位区域时逐步揭露底层页脚。
  - **运动规律**：`Footer` 使用 `position: fixed` 锚定视口底部，`FooterSpacer` 在文档流中腾出等高空间，上方主内容滚走后逐渐"揭开"底层。
  - **视觉**：巨型文字层锚定在视口底部固定坐标系，可视面积随遮罩层上移而增加
  - **过渡保护**：开信切换到主体验期间，会额外以 `isTransitioning` 全屏遮罩覆盖页脚，避免底层页脚在过渡中提前露出。
- **结构**（自上而下）：
  1. **大页脚导航 (Footer Navigation)**：红色背景 `#C23643`，4 张链接卡片横向排列
     - QQ 交流群（用于活动通知/反馈收集）
     - 项目主页（用于站点主入口跳转）
     - GitHub 仓库（用于源码与更新说明）
     - 邮件联系（用于合作或版权联络）
  2. **巨大化 HIMEMATSU Logo**：使用 `public/HIMEMATSU.svg`，响应式占满网页宽度，红色背景
  3. **次页脚版权信息 (Sub Footer)**：红色背景条
     - 左侧：`Copyright © 2026 HIMEMATSU. All rights reserved.`
     - 右侧：`琼ICP备114514号`
- **配色**：背景 `#C23643`，文字 `#ECE9E4`（90% 不透明度用于次级文字）

## 5. 视觉风格 (Visual Style)

### 5.1 关键词
- **Galgame**: 精致人物立绘，情感化叙事。
- **Skeuomorphism (拟物)**: 信封、火漆印、邮票齿孔、纸张纹理。
- **Glassmorphism (玻璃拟态)**: 模态框背景采用高斯模糊 (Backdrop Blur)。

### 5.2 调色板 (Palette)
> 详见 `docs/design/视觉规范.md` 的 §2.1 色彩体系。以下为快速参考：
- **Primary**: `#c23643` (暗红/情书红)
- **Background**: `#ece9e4` (暖灰纸面)
- **Surface**: `#f6f1eb` (纸面卡片)
- **Ink Strong**: `#342628` (标题深墨色)
- **Ink**: `#5a4748` (正文默认色)

## 6. 创作公示板 `/creation` (Creation Board)

> 详细方案见 [创作公示板](./创作公示板.md)。

### 6.1 页面目标

- 用于集中展示群友们已公开的共创灵感。
- 让后续创作可以直接站在已有灵感之上继续补充，而不是从零散聊天记录中重新翻找。

### 6.2 页面结构

- 顶部为标题区，仅承载标题与一句简短说明。
- 标题区下方直接进入便签式瀑布流，不再引入多余分栏、侧边栏或复杂筛选面板。
- 页面主体以“轻网格草稿线背景 + 便签卡片瀑布流”构成。

### 6.3 卡片与布局

- 灵感内容以便签条样式展示，卡片高度随内部堆叠内容自然变化。
- 卡片底色为随机马卡龙色系，每张卡片一个色调。
- 卡片外轮廓为直角，不使用圆角。
- 列表采用瀑布流布局，形成创作墙般的高低错落效果。
- 一张卡片对应一个 `CardID`，同 `CardID` 的多条记录在卡片内部垂直堆叠。
- 卡片右上角固定展示常驻的白色 `新增创意` 按钮，按钮需预留独立空间，点击后打开带当前 `CardID` 预填值的飞书表单。
- 每条堆叠项的左上角为 `author`，右上角为胶囊 `tags`，主体根据标签展示 `content` 或 `images`。

### 6.4 交互原则

- 默认以扫读为主，点击后再进入完整阅读。
- 动效应轻盈克制，以抬升、归正、阴影变化为主，不做强弹跳。
- 手机端优先保留内容可读性，必要时缩减列数，不强求桌面端的展板密度。
