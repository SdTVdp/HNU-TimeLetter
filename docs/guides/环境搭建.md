# 环境搭建与开发流程

## 1. 安装依赖
确保 Node.js 版本 >= 20。
```bash
npm install
```

## 2. 环境变量配置
复制 `.env.example` 到 `.env.local` 并填入凭证：
```bash
cp .env.example .env.local
```

关键环境变量（详见 [飞书 CMS 配置](./飞书CMS配置.md)）：
- **飞书基础**：`FEISHU_APP_ID` / `FEISHU_APP_SECRET` / `FEISHU_APP_TOKEN` / `FEISHU_TABLE_ID` / `FEISHU_VIEW_ID`
- **创作公示板（可选，未配置则用内置默认值）**：`FEISHU_CREATION_TABLE_ID` / `FEISHU_CREATION_VIEW_ID`
- **阿里云 OSS**：`ALIYUN_OSS_REGION` / `ALIYUN_OSS_BUCKET` / `ALIYUN_OSS_ACCESS_KEY_ID` / `ALIYUN_OSS_ACCESS_KEY_SECRET`
- **后台鉴权**：`ADMIN_PASSWORD`（用于 `/admin` 登录，未配置时回退到 `admin`，上线前务必覆盖）

## 3. 数据同步 (关键步骤)
项目**不包含** Mock 数据，必须连接飞书同步真实数据：
```bash
npm run sync
```
*成功后会在 `src/data/` 下生成 `content.json`（首页故事）与 `creation-board.json`（创作公示板）。*

## 4. 启动开发服务器
```bash
npm run dev
```
访问 `http://localhost:3000`。

## 5. 安装与使用 Playwright
项目已集成 Playwright 端到端测试工具。首次在本地使用时，安装 Chromium 浏览器：
```bash
npm run test:e2e:install
```

常用命令：
- `npm run test:e2e`: 无头模式运行 E2E 测试
- `npm run test:e2e:headed`: 有界面模式运行 E2E 测试
- `npm run test:e2e:ui`: 打开 Playwright UI 调试测试

默认配置会自动启动本地开发服务器 `http://127.0.0.1:3000`，测试目录为 `tests/e2e/`。

### 5.1 Playwright MCP 工具接入
仓库根目录提供了项目级 [opencode.json](../../opencode.json)，用于为 OpenCode 注册 `playwright` MCP server，避免只依赖个人全局配置。

首次验证时执行：
```bash
opencode mcp list
```

期望看到类似输出：
- `playwright connected`

补充说明：
- 当前配置会使用 `npx -y @playwright/mcp@latest --headless --output-dir .playwright-mcp` 启动 MCP server。
- 调试产物会写入仓库根目录的 `.playwright-mcp/`，该目录应保持本地忽略，不提交到版本库。
- 如果 `opencode.json` 是在当前代理会话启动后才新增或修改的，需要重开一次 OpenCode 会话，MCP 工具列表才会重新注入。
- 如果 `opencode mcp list` 没有显示 `playwright connected`，优先检查本机 Node.js、`npx` 网络访问，以及是否在项目根目录启动 OpenCode。

## 6. 协同开发规范 (Workflow)

### 6.1 分支管理 (Git Branching)
- `main`: **生产分支** (Vercel 自动部署)。仅允许通过 PR 合并，禁止直接 Push。
- `dev`: **开发主分支**。所有功能分支合并至此进行联调。
- `feat/infra`: Dev A 工作分支。
- `feat/desktop`: Dev B 工作分支。
- `feat/mobile`: Dev C 工作分支。

### 6.2 目录权限 (Directory Ownership)
为了防止代码冲突，请严格遵守以下目录修改权限：
- `src/app/`: [Dev A] 全局路由与布局
- `src/components/desktop/`: [Dev B] 桌面端专用组件
- `src/components/mobile/`: [Dev C] 移动端专用组件
- `src/components/shared/`: [Dev A] 通用组件
- `src/lib/`: [Dev A] 类型定义与工具函数

### 6.3 编译边界 (Compile Boundary)
- `agents/` 与 `.opencode/agents/` 属于协作代理资产，不参与前端应用运行时编译。
- 项目已在 `tsconfig.json` 中排除 `agents/`，用于避免代理知识库和工具文件影响 `npm run build`。
- 新增 Agent / Skill / 文档类资产时，请继续放在约定目录内，不要混入 `src/`。

## 7. 常见命令
- `npm run build`: 生产构建 (会自动执行 sync)。
- `npm run lint`: 代码检查。
- `npm run test:e2e`: 运行 Playwright 端到端测试。
